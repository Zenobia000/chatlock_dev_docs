# BR-M18-02 — Change request process

## Rule

Configuration changes 需要 request、owner approval、effective_date、rollback note。approval 後依 BR-M18-04 staged policy（5%/50%/100%, 每段 30 min observation, 符合 fast-track 三條件可降至 15 min）執行 rollout。

## Source

- docs/_source/01-workorder-erp.md §M18 System Admin（BR-M18-02 + G039）
- value-decisions 2026-05-28（rollout timing alignment）

## Constraints

- 4 必填欄位（request / approval / effective_date / rollback note）
- effective_date 不可早於 approval 時間
- high-risk 變更（pricing / RBAC / SLA / cancellation / refund）不可走 fast-track

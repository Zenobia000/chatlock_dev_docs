# BR-A01-02 — Reply token 限制

## Rule

AI 回覆 token 上限 = **1500 tokens（中文約 1000 字）**。超過時 guardrails 必須 truncate 並補「（詳情請洽客服）」尾段，不可發空訊息。

## Source

- docs/_source/02-ai-chatbot-sync.md §A M01 進線 Debounce
- value-decisions 2026-05-28（業主裁決）

## Rationale

LINE chatbot 客戶超過 200 字會掃讀 bounce，1500 tokens 剛好涵蓋 RAG 詳細答覆又不切話。優於 subagent 推的 2048（過長）與舊預設 1024（RAG 答覆容易切尾）。

## Constraints

- token cap = 1500 (hard limit at guardrails layer)
- truncation marker required when cut（避免半句話送出）
- configurable via M18（rollout 走 BR-M18-04）

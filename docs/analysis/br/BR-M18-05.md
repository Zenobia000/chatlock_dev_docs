# BR-M18-05 — Config audit retention ≥ 7 年

## Rule

每筆 config change 必須記錄 who / when / what diff / why，retention ≥ 7 年（合規 + 商業會計法）。staged rollout 的每段 promotion（5% → 50% → 100%）與 fast-track 觸發條件皆需獨立 audit row。

## Source

- ADR-0067 §audit；商業會計法 §83
- value-decisions 2026-05-28（rollout audit granularity）

## Constraints

- append-only ledger
- retention ≥ 7 year
- 4 必填欄位（who / when / diff / why）
- 每段 stage promotion 一 row（5/50/100 至少 3 rows / change）

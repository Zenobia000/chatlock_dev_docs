# BR-M18-04 — Config 變更 staged rollout

## Rule

高風險 config 變更必須採 **3 段 staged rollout = 5% → 50% → 100%**，每段 observation window = **30 min**，rollback budget ≤ 1 min。

**Fast-track = 15 min** 適用條件（AND）：
1. schema validation pass（pre-deploy 自動驗證）
2. 5% canary 階段觀察期 0 error
3. 變更類別不在 high-risk allowlist（pricing / RBAC / SLA）

## Source

- ADR-0067 §staged_rollout
- value-decisions 2026-05-28（業主裁決）

## Rationale

萬級用戶 SaaS 不需 Google 規模 10 min canary，但 config 改錯直接影響業務。30 min 才夠涵蓋 ops alarm cycle（5-10 min）+ 客服回饋 cycle（10-20 min）。Fast-track 給已 schema-validated + 0-error 的低風險變更，避免每次都壓 90 min 卡 hotfix。

## Constraints

- rollout stages: [5%, 50%, 100%]（不可少於 3 段）
- observation window 預設 30 min / stage，high-risk 強制 standard mode
- fast-track 15 min / stage 僅當 3 條件 AND 成立
- rollback budget ≤ 1 min（automated revert）
- high-risk allowlist：pricing / RBAC / SLA / 取消費 / 退款門檻
